# Snapshot の仕様

## Snapshot の命名規則

- フォーマット:
    - `Snapshot No.<Number>`
- `No.<Number>` は過去の Snapshot に付与されている値からインクリメントし、重複しないようにしてください
    - e.g. No.1 → No.2
    - 過去の Snapshot が見つからないときは No.1 からスタートして下さい
- Snapshot 名に 『YYYY-MM-DD』 のような具体的な現実日時を情報として埋め込まないでください 
    - e.g. Snapshot 2025-01-01 ← NG (必ず `Snapshot No.<Number>` 形式を使用する)

## Snapshot 作成後の後処理

- 最新の Snapshot 作成後は、古い Snapshot をパージし、参照しないでください。
    - 理由: 複数の Snapshot を参照することで時系列が混線してしまう可能性を抑止します

## Snapshot に含まれる確定状況の項目とその概要

1. `時期感・季節`:
    - Snapshot 作成時点の時期感と季節。
    - 厳密な日時は必要なく、曖昧で良い。 (e.g. 1月末頃、新春 等)
2. `このシーンで外部観察された事実の概要`:
    - 登場人物の行動・発話から、どういうやり取りがおこなれたのかを分析し、客観性を維持したまま概要を作成する。
    - 登場人物の内面の推定は一切必要ない。
3. `ひよりの状況（Status）`:
    - このシーンでひよりに、何がおきて、どのような重要な行動・発話が観測されたか。
    - ひよりの内面の推定は一切必要ない。
4. `小宮の状況（Status）`:
    - このシーンで小宮に、何がおきて、どのような重要な行動・発話が観測されたか。
    - 小宮の内面の推定は一切必要ない。
5. `ひよりと小宮の間の約束`:
    - このシーンで次のシーンに接続（申し送り）する必要がありそうな、重要な約束があれば記録する。
    - 約束がなければ、空白で問題ない。
    - e.g. 一緒に買い物に行く約束。特定の話題について話をする約束 等

# Snapshot の目的

- Snapshot 作成の目的:
    - シミュレーション空間で発生している最新の「事実」を正しく保存する
- Snapshot を何に使用するか:
    - あなた（システム）がシミュレーション空間を誤解少なく運用するために参照する（直近のシミュレーション空間のコンテキスト）
- シミュレーション空間の登場人物は Snapshot を認知できない:
    - Snapshot はメタ空間で作成し、あなた（システム）が利用するデータです。
    - シミュレーション空間の登場人物（人格モデル）は、Snapshot を認知してはいけません。
